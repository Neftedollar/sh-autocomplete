Why shac

Built for daily shell muscle memory, not demo magic.

Finish commands faster

Use partial command input like `git ch`, `kubectl con`, or `pyt`, then let `shac` surface the likely next move.

Keep your data local

Config, SQLite data, caches, and daemon state stay on your machine. The daemon and shell adapters do not call out to the network.

Understand the ranking

`shac explain` lets you inspect the feature contributions behind a suggestion instead of treating autocomplete as a black box.

Quickstart

Useful in minutes, not after a weekend config binge.

The current MVP is strongest on `zsh`, with `bash` supported as best-effort list completion. Start the daemon, install the adapter, and reindex once.

Full quickstart Diagnostics Privacy 
cargo build --bins
./target/debug/shac install --shell zsh --edit-rc
./target/debug/shac daemon start
./target/debug/shac reindex
./target/debug/shac debug completion --shell zsh --line "git ch" --cursor 6
./target/debug/shac explain --shell zsh --line "kubectl con" --cursor 11
