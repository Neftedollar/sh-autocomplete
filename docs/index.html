Why shac

Built for daily shell muscle memory, not demo magic.

Works from day one

Import your zsh history, zoxide destinations, and git projects on `shac install`. Useful completions appear before you've typed a single new command.

Smarter `cd`

`cd ` surfaces your most-visited directories (shown with a `→` prefix) alongside the current folder's children, ranked by frecency.

Always up to date

The daemon indexes command docs in the background every 6 hours. New tools appear in completions automatically.

Keep your data local

Config, SQLite data, caches, and daemon state stay on your machine. The daemon and shell adapters do not call out to the network.

Self-documenting

shac shows context-aware feature hints in the completion menu — git branches when you're in a repo, ssh hosts when you start `ssh `, npm scripts when there's a `package.json`. Run `shac suggest` for a context-aware list of what shac can do here. Tips auto-mute after a few showings; disable globally with `shac config set ui.show_tips false` or `SHAC_NO_TIPS=1`.

Quickstart

Useful in minutes, not after a weekend config binge.

The current MVP is strongest on `zsh`, with `bash` supported as best-effort list completion. Start the daemon, install the adapter, and reindex once.

Full quickstart Diagnostics Privacy 
brew install Neftedollar/shac/shac
shac install --shell zsh --edit-rc
brew services start shac
shac reindex
shac doctor
shac invalidate-caches   # flush dir cache
shac reindex --full      # re-index all command docs
